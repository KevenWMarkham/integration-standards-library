Module Highlights

ISL-03: Naming Conventions
↓ 20-32 hrs

7 naming templates covering database, table, column, pipeline, API, and infrastructure naming. Fabric-specific patterns (lh_, wh_, pl_, nb_, sm_ prefixes) included. Eliminates subjective naming debates.

ISL-04: Data Classification
↓ 35-60 hrs

4-tier classification model (Public/Internal/Confidential/Restricted) with Purview label alignment, compliance mapping (SOX, GDPR, ITAR), and manufacturing overlays for trade secrets and IoT/OT data.

ISL-01: API Governance
↓ 60-85 hrs

Complete API lifecycle governance: design standards, versioning policy, OAuth 2.0/OIDC security, rate limiting, and catalog requirements. OWASP API Top 10 cross-referenced.

ISL-06: Data Quality
↓ 30-45 hrs

50+ pre-built quality rules across schema, row-level, aggregate, and business categories. SLA thresholds per classification tier. Quality scorecard with RAG status.

ISL-02: Metadata & Lineage
↓ 70-110 hrs

Business glossary, technical metadata schema (Purview-aligned), column-level lineage requirements, catalog governance. Manufacturing extensions for IoT sensor registries and ERP cross-references.

ISL-05: Integration Patterns
↓ 60-90 hrs

8 reusable patterns: ERP extract, IoT ingestion, API gateway, event-driven, MDM sync, file-based, medallion architecture, reverse ETL. Decision framework for pattern selection.
